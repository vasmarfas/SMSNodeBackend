# SMSNodeBackend

**English:** [`README.en.md`](README.en.md)

## Назначение

Асинхронный сервер для маршрутизации SMS и работы с GSM-шлюзами: REST API (FastAPI), Telegram-бот, очередь отправки SMS в процессе, адаптеры GOIP/Skyline (UDP и HTTP), хранение в PostgreSQL.

## Данные при развёртывании

**Тестовые или демонстрационные записи при запуске контейнера или `main.py` не создаются автоматически.** Поднимается только схема БД (таблицы) через `init_db` / `create_tables`. Шлюзы, SIM, пользователи и сообщения нужно добавить через API или бота.

### Демо-режим (Read-Only)

Если в `.env` (или окружении) установить `IS_DEMO=True`, при запуске бэкенд:
1. Автоматически создаст тестовые данные на русском и английском языках.
2. Создаст учётные записи: `demo` (пароль: `demo`, права пользователя) и `admin` (пароль: `admin`, права администратора).
3. Создаст моковые шлюзы (UDP и HTTP) и сгенерирует сообщения.
4. **Включит мягкий Read-Only режим**: все операции записи, изменения и удаления данных (через API и Telegram-бот) будут успешно обрабатываться логикой и отдавать клиенту правильные успешные ответы, но **без фактического сохранения в базу данных** (транзакции откатываются в конце запроса). Это позволяет тестировать весь UI без накопления "мусора" в демо-базе.

Отдельный сценарий **`seed_fake_data.py`** — вспомогательный скрипт, который вызывается в демо-режиме, но также может быть запущен вручную:
```bash
py -3 seed_fake_data.py
```

## Быстрый запуск

### 1. Подготовка окружения
```bash
cp .env.example .env
```
В файле `.env` обязательно задайте `BOT_TOKEN`, `ADMIN_ID`, `API_SECRET_KEY` и параметры PostgreSQL (если не используете Docker Compose с базой).

### 2. Запуск через Docker (рекомендуется)
Вы можете собрать образ локально или использовать готовый из GitHub Container Registry (GHCR).

**Вариант А: Сборка из исходников (по умолчанию)**
```bash
docker compose up -d --build
```

**Вариант Б: Готовый образ из GHCR**
Если вы не хотите собирать образ локально (или у вас слабый сервер), раскомментируйте строку `DOCKER_IMAGE` в `.env` и укажите путь к образу:
```env
DOCKER_IMAGE=ghcr.io/vasmarfas/smsnodebackend:latest
```
После этого запустите:
```bash
docker compose up -d
```

### 3. Локальный запуск (без Docker)
Убедитесь, что у вас запущен сервер PostgreSQL. Настройки подключения (`POSTGRES_HOST`, `POSTGRES_PORT` и т.д.) указываются в `.env`.

```bash
python -m venv .venv
source .venv/bin/activate  # На Windows: .venv\Scripts\activate
pip install -r requirements.txt
python main.py
```

API по умолчанию: `http://localhost:8000` (или порт из `API_PORT` в `.env`). Проверка: `GET /health`.

## Эксплуатация

- Для production ограничьте CORS, не оставляйте `allow_origins=["*"]` без необходимости.
- Используйте надёжный `API_SECRET_KEY`.
- Bcrypt обрезает пароль до 72 байт; политика паролей на клиенте должна это учитывать.

## Тесты

```bash
set RUN_API_TESTS=1
pytest tests/
```

Часть тестов не выполняется без переменной `RUN_API_TESTS`.
